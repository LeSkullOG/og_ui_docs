List Detail UI ​

OpenList opens a reusable list/detail interaction surface for dense RP tools such as player lists, job rosters, dispatch calls, invoices, licenses, admin panels, and documents.

lua
CreateThread(function()
    local result = exports.og_ui:OpenList({
        title = 'Staff Tools',
        subtitle = 'Online player actions',
        searchable = true,
        items = {
            {
                label = 'Jake Stone',
                subtitle = 'ID 12 - Police',
                value = 12,
                icon = 'JS',
                category = 'Police',
                priority = 'High',
                priorityType = 'danger',
                badge = 'On Duty',
                badgeType = 'success',
                status = 'Responding',
                statusType = 'warning',
                time = '2m ago',
                description = 'Senior officer currently assigned to patrol duties.',
                meta = {
                    { label = 'Job', value = 'Police' },
                    { label = 'Grade', value = 'Sergeant' }
                },
                actions = {
                    { label = 'Message', value = 'message', variant = 'primary' },
                    { label = 'Inspect', value = 'inspect' }
                }
            }
        }
    })

    if result and result.ok then
        print(('Record: %s, Action: %s'):format(result.value, result.actionValue))
    end
end)

Result ​

lua
{
    ok = boolean,
    cancelled = boolean,
    type = 'list',
    id = string,
    item = table | nil,
    action = table | nil,
    value = any,
    actionValue = any
}

List Fields ​

FieldTypeNotes
itemstableArray of records.
searchablebooleanSet false to hide search.
showCategoriesbooleanSet false to hide category filters.
emptyTextstringMessage shown when no records match.
detailTitlestringOptional detail panel title.
selectTextstringDefault action label when a record has no custom actions.
loadingbooleanShows the loading state instead of records.
statestringSet to loading or error for explicit state rendering.
errorstring/tableError message, or table with message, shown in the error state.
emptyTitlestringCustom title for the empty state.
emptyTextstringCustom description for the empty state.
loadingTextstringCustom title for the loading state.
errorTextstringCustom fallback error message.

Item Fields ​

FieldTypeNotes
labelstringMain row title.
subtitlestringSecondary row text.
valueanyStable identifier returned in result.value.
iconstringText fallback shown in the avatar area.
imagestringOptional image URL/path.
categorystringOptional category/search text.
prioritystringOptional urgency badge, such as High, Medium, or Low.
priorityTypestringOptional tone override for priority. Suggested values: success, warning, danger.
badgestringOptional status badge.
badgeTypestringSuggested values: default, success, warning, danger, info.
statusstringOptional second state badge and detail metadata row.
statusTypestringOptional tone for status. Suggested values: default, success, warning, danger, info.
timestringOptional compact timestamp shown in the row, badges, and detail metadata.
timestampstringAlias-style timestamp field if that better matches your backing data.
descriptionstringDetail panel description.
metatableDetail rows shaped as { label = 'Job', value = 'Police' }.
metadatatableObject or array alternative to meta; useful for API-shaped records.
disabledbooleanDisabled rows cannot be selected.
actionstableOptional item actions. Defaults to a single select action.

For local testing, /oglist opens a staff-tool style list/detail panel with priority, status, timestamp, and metadata examples.

Use /ogstates to preview empty, loading, and error list states.
