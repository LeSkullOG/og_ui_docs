Grid UI ​

OpenGrid opens a reusable slot-grid interaction surface. It is intended as a UI face for inventories, shops, crafting benches, lockers, evidence storage, wardrobes, and similar systems.

OG_UI does not own item movement or inventory state. It returns the selected item and action so your resource can decide what happens next.

lua
CreateThread(function()
    local result = exports.og_ui:OpenGrid({
        title = 'Crafting Bench',
        subtitle = 'Available recipes',
        searchable = true,
        columns = 5,
        lockMovement = true,
        captureMouse = true,
        keepInput = false,
        items = {
            {
                label = 'Lockpick',
                value = 'lockpick',
                description = 'A small tool used for delicate locks.',
                icon = 'LP',
                count = 3,
                weight = 0.1,
                rarity = 'common',
                category = 'Tool',
                actions = {
                    {
                        label = 'Craft',
                        value = 'craft',
                        variant = 'primary'
                    },
                    {
                        label = 'Inspect',
                        value = 'inspect'
                    }
                }
            }
        }
    })

    if result and result.ok then
        print(('Item: %s, Action: %s'):format(result.value, result.actionValue))
    end
end)

Result ​

lua
{
    ok = boolean,
    cancelled = boolean,
    type = 'grid',
    id = string,
    item = table | nil,
    action = table | nil,
    value = any,
    actionValue = any
}

Grid Fields ​

FieldTypeNotes
itemstableArray of slot items.
columnsnumberNumber of grid columns. Defaults to 5.
searchablebooleanSet false to hide search.
headingstringOptional large heading inside the grid panel.
gridTitlestringOptional title above the slot grid.
detailTitlestringOptional title above the detail panel.
selectTextstringDefault action label when an item has no custom actions.
minSlotsnumberPads the grid with empty slots up to this count.
slotsnumberAlias for minSlots.
showCategoriesbooleanSet false to hide category filters.
loadingbooleanShows the loading state instead of slots.
statestringSet to loading or error for explicit state rendering.
errorstring/tableError message, or table with message, shown in the error state.
emptyTitlestringCustom title for the empty state.
emptyTextstringCustom description for the empty state.
loadingTextstringCustom title for the loading state.
errorTextstringCustom fallback error message.

Item Fields ​

FieldTypeNotes
labelstringVisible item name.
valueanyStable item/action identifier returned in result.value.
descriptionstringDetail panel description.
iconstringText fallback shown when no image is supplied.
imagestringOptional image URL/path for the item visual.
countnumberOptional quantity shown in the slot corner.
weightnumberOptional weight shown in the slot/detail metadata.
durabilitynumberOptional 0-100 durability bar.
qualitynumberAlias-style 0-100 quality bar when durability is not supplied.
conditionnumberAlias-style 0-100 condition bar when durability/quality are not supplied.
raritystringSuggested values: common, uncommon, rare, epic, legendary.
categorystringOptional metadata/search text.
equippedbooleanShows an equipped badge and stronger slot cue.
lockedbooleanShows a locked badge and danger slot cue.
hotbarnumber/stringShows hotbar assignment on the slot and detail panel.
badgestringOptional custom status badge in the detail preview.
badgeTypestringSuggested values: default, success, warning, danger, accent.
statusstringOptional detail status badge.
statusTypestringOptional tone for status.
typestringOptional detail metadata row.
serialstringOptional detail metadata row.
ownerstringOptional detail metadata row.
metadatatableOptional extra detail rows. Supports key/value tables or { label, value } arrays.
disabledbooleanDisabled slots cannot be selected and are skipped by controller navigation.
actionstableOptional item actions. Defaults to a single select action.

For local testing, /oggrid opens a fake crafting-style grid.

Use /ogstates to preview empty, loading, and error grid states.
